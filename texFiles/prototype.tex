\pagebreak \section*{5. Revised Features}
\addcontentsline{toc}{subsection}{Features}
We consolidated the features list into a clearer selection for the purposes of our prototype. Features were broken down into 'demo pages'. All sections will have a navigation bar at the bottom to navigate between moodle, the home page and an inbox integrated with a user's email account.\\


\textbf{Home Page}\\
What features we considered most important to access upfront. Access to info such as university news or services would be placed into their own dedicated widgets.
\begin{itemize}
  \item \textit{News}\\
        Latest updates and relevant information pertaining to the university.
  \item \textit{Clubs \& Societies}\\
        Featured clubs and societies, if a user was logged in, display presently joined clubs/societies, particular updates to those clubs or societies within this widget.
  \item \textit{Supports \ Serviecs}\\
        Allow users to access university support services/utilities, grants \& fee management. 
  \end{itemize}


\textbf{Navigation Pane}\\
Easy navigation across all site features.
\begin{itemize}
  \item \textit{Academic Calendar}\\
      Relevant university events, allow editing for individual students to add their own information similar to a study planner.
  \item \textit{Course Timetable}\\
         Timetable provided based on the details of a logged in user, shows various module assignment dates, exams, etc.
  \item \textit{News \& Events}\\
        Expanded News and Events section for subjects relating to university or associated clubs \& society events.
  \item \textit{To-Do List/Notes Tool}\\
         Utility for quick notes, tracking of various deadlines set by Moodle, templates for study schedules.
  \item \textit{Transport Schedules}\\
        Bus/train commute info, could be integrated with services such as TFI.
  \item \textit{Library}\\
        Options to reserve/renew loaned books, organise study pods, check opening hours, etc.
  \item \textit{Campus Map}\\
        Integrated Google Map of the SETU campus, options to add custom markers, notes, etc. to buildings.
   \item \textbf{Support Services} \\
        University support services/utilities, grants \& fee management. 
  \end{itemize}


\textbf{Built-In Calendar}\\
Full, user-specific customisable calendar utility, 
\begin{itemize}
  \item \textit{Semester Period}\\
        Display current semester and exam period, allow user to look ahead to future months or review progress in previous.
  \item \textit{Event Maker}\\
        Allow users to add their own events and colour code as preferred.
  \item \textit{Offline Access}\\
        Save calendar events and schedule to user device.
  \end{itemize}
  
  
\textbf{Course Timetable}\\
Review SETU Timetable in relation to logged-in user course, if applicable.
\begin{itemize}
  \item \textit{Timetable Details}\\
        Display user programme, student group and department. If user is a guest, possible to display an orientation days select events instead.
  \item \textit{Weekly Timetable}\\
        Timetable colour coded by lecture or lab, updated automatically in sync with similar purpose website SETU timetable.
  \item \textit{Offline Access}\\
        Save or adapt existing timetable by user session. 
  \end{itemize}


\textbf{News \& Events}\\
     Informational section for subjects relating to university or associated clubs \& society events.
\begin{itemize}
  \item \textit{Search}\\
        Search for articles or events by specific keywords, headings, etc.
  \item \textit{Filter}\\
       Filter by news/article category, e.g. academic, society related, etc.
  \item \textit{Real-time Notifications}\\
       Allow users to opt in to following particular events/articles 
  \end{itemize}

\textbf{To-Do Notes}\\
     Utility for quick notes, saved to user device.
\begin{itemize}
  \item \textit{Search}\\
        Search for notes by specific keywords, headings, etc.
  \item \textit{Filter}\\
       Filter by colour coding, subject, etc.
  \item \textit{Moodle Integration}\\
       Allow relevant Moodle notes to be saved to user's to-do list.
  \end{itemize}

\textbf{Settings}\\
     Various accessibility, notification and general preference user settings.
\begin{itemize}
  \item \textit{Vision Accessibility}\\
       Low vision, colour blind and screen reader support settings.
  \item \textit{Notification}\\
       Enable push notification, select what kinds of notifications to allow and/or enable Do Not Disturb (DND) between specific hours.
  \end{itemize}


\textbf{Integrated Inbox}\\
    Allow users to connect several email accounts, e.g. outlook, gmail, to the one inbox.
\begin{itemize}
  \item \textit{Sort}\\
        Search for notes by specific keywords, headings, etc.
  \item \textit{Filter}\\
       Filter by colour coding, subject, etc.
  \item \textit{Moodle Integration}\\
       Allow relevant Moodle notes to be saved to user's to-do list.
  \end{itemize}


\textbf{User Experience}\\
Overall considerations for user experience remains unchanged from initial features list.
\begin{itemize}
  \item \textit{Intuitive Design}\\
        Menus and layout should be concise and visually clear. Integrated first or third-party features should not be unnecessarily bloated or redundant.
  \item \textit{Cross-Platform Sync}\\
        Sync a user across devices, (desktop, phone, etc.) and the SETU website when internet is available.
  \item \textit{Offline Access}\\
        Allow users to save info such as timetables, lab results, etc. for use offline.
  \item \textit{Feedback \& Reports}\\
        Users should be able to report bugs, criticism or suggestions on improving the app.
  \item \textit{Accessibility Tools}\\
       Settings for low-vision, colour-blind, screen-readers, etc. 
  \item \textit{Free and Efficient}\\
       Assuming development is supported by SETU, advertisements or paid premium memberships should not be expected. Exceptions for society memberships, student tuition fees.
  \item \textit{Real-time Notifications}\\
       Users may opt-in to various push notifications, updates, etc.
   \item \textit{GDPR compliant; data secure}\\
       Use of 2FA, privacy options, secure cloud storage handled by services like Cloudflare, Google Drive. Payment services also trusted to a third party such as stripe/google pay.
  \end{itemize}

A prototype is an early example or model of a product used to view and test visual concepts, processes, and design approaches. Alterating, iterating and demonstrating features at this stage of development is significantly less costly on project resources, and allows for better communication between developers and clients.

\subsection*{Personal Reflections}
\subsection*{Madison Roche 20112525}
My most significant contribution to the project were the JUnit Tests and the Prototype examples. We shared most roles between the four of our team members but as we neared the end of the project we took on separate parts to speed up the ending process.\\

One challenge we faced was dealing with time management issues. Although the deadlines are quite spread out usually having a week or two between each, we still found ourselves leaving most sections until the day before to fully flesh out and upload. We started to make our own deadlines so we could have the work done a couple of days before hand, leaving a day or two before the deadline to add or clean-up any issues or mistakes with the documents. \\

One important process decision we made was to divide the work more clearly toward the end of the project, with each team member taking over a certain part of the project. This helped speed our process up and made the final stages of the document more efficient. It did mean that we had less collaboration overall which can cause limited understanding but allows us to work on what we know best.\\

In a future project, I would focus on keeping a consistent chain of progress from the beginning rather than leaving it fall behind and become work for the day before. Even though we introduced earlier deadlines for ourselves, starting tasks sooner than later and spreading the workload more evenly may have reduced any worries and could've improved the quality of the work even more. I think it'd also allow for more refinement of other sections and parts too.\\
